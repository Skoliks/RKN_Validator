# Чек-лист соответствия сайтов и доменов
**Дедлайн: 1 сентября 2026.**

---

## ЧАСТЬ A. Идентификация доменов через Госуслуги (569-ФЗ от 29.12.2025)

Действует **только для зон .ru / .рф / .su**. Без идентификации администратора через ЕСИА после 1 сентября нельзя: зарегистрировать новый домен, продлить, сменить администратора, перенести к другому регистратору, изменить NS-серверы.

### Базовые шаги (по каждому домену)
- [ ] **Инвентаризация.** Свести все домены в таблицу: домен, зона, регистратор, на кого оформлен администратор (ФИО/юрлицо), дата окончания, на чьём аккаунте ЕСИА будет идентификация, статус.
- [ ] **Проверить администратора.** Кто юридически владелец (не технический специалист, а лицо в карточке домена). Особое внимание — домены на бывших сотрудниках, подрядчиках, старых юрлицах.
- [ ] **Проверить ЕСИА.** У нужного лица должна быть **подтверждённая** учётная запись на Госуслугах. Упрощённая (по телефону/почте) не подойдёт.
- [ ] **Сверить данные.** Данные администратора в ЛК регистратора и в ЕСИА должны совпадать (ФИО, дата рождения, ИНН, контакты; для юрлиц — наименование, ИНН, ОГРН, адрес). Расхождения → отказ. Актуализировать заранее.
- [ ] **Пройти идентификацию** в ЛК регистратора: «пройти идентификацию» → авторизация в ЕСИА → выдача регистратору прав на сверку. Пароль от Госуслуг регистратор не видит.
- [ ] **Если домен на другое физлицо** — регистратор даёт ссылку, по которой это лицо идентифицируется само.
- [ ] **Не откладывать на август** — нагрузка на ЕСИА и регистраторов, часть проверок вручную (от 1 рабочего дня).

### Особые случаи доменов

**Домен в нероссийской зоне (.com / .io / .net и т.п.)**
- [ ] 569-ФЗ **не применяется**. Идентификация через Госуслуги не нужна. Остальные требования части C — действуют независимо от зоны.

**.ru/.рф/.su у «зарубежного» регистратора**
- [ ] Эти зоны обслуживают только российские аккредитованные регистраторы; иностранный «регистратор» = реселлер поверх российского. Если реселлер не даёт ЕСИА-идентификацию → **перенести домен к российскому аккредитованному регистратору до 1 сентября.**

**Администратор — иностранное лицо/компания**
- [ ] Иностранцу тоже нужна подтверждённая ЕСИА + идентификация (процедура для иностранных граждан существует). Если нереалистично — **сменить администратора** на российское лицо/юрлицо до 1 сентября.

**Домен на физлицо, фактически пользуется компания**
- [ ] Законно. Идентификацию проходит физлицо-администратор по своей ЕСИА.
- [ ] Корпоративный риск: домен принадлежит человеку, не компании. Решение — либо **сменить администратора на компанию** (пока оба существуют — простая онлайн-процедура, обработка до 3 раб. дней), либо закрепить пользование договором.

**Юрлицо-администратор ликвидировано**
- [ ] **До ликвидации** — обязательно передать права действующему лицу (смена администратора). Многократно проще.
- [ ] **После ликвидации** — процедура «перерегистрация при прекращении деятельности администратора»: аннулирование + перерегистрация на нового. Нужны лист записи о ликвидации / выписка ЕГРЮЛ. При банкротстве <1 года — нотариальные согласия бывших участников об отказе от прав. Услуга платная.
- [ ] Срочность: регистратор **вправе аннулировать** домен на ликвидированной компании (ст. 419 ГК + Правила регистрации). Действовать до аннулирования.

**«Аренда» / безвозмездное пользование доменом**
- [ ] Домен — право администрирования, не объект вещного права; классическая «аренда» неприменима. Рабочая схема: администратор прежний (физлицо), компания пользуется по договору (возмездного/безвозмездного пользования или услуг).
- [ ] Для 569-ФЗ значим именно администратор — он проходит идентификацию и несёт ответственность за контент. Если нужно «на компанию» — это смена администратора, не аренда.
- [ ] Безвозмездное пользование может дать налоговый вопрос (внереализационный доход у выгодоприобретателя) — к бухгалтеру.

## ЧАСТЬ B. Авторизация пользователей (199-ФЗ от 26.06.2026; требование — ч.10 ст.8 149-ФЗ, действует с 01.12.2023)

Штрафы по новой ст. 13.55 КоАП с 1 сентября 2026: юрлица 500–700 тыс. ₽, повтор до 1,4 млн. Адресат — владелец сайта/ИС/приложения, не пользователи.

### Что убрать
- [ ] Снять кнопки **«Войти через Google» / «Войти через Apple»** (Google Sign-In, Apple ID) и иные иностранные OAuth (GitHub/Microsoft login).
- [ ] Виджет **«Войти через Telegram»** на сайте — заменить (Telegram не в перечне разрешённых систем).

### Что оставить/внедрить (разрешённые способы)
- [ ] **Российский номер телефона** (SMS/звонок) — способ №1.
- [ ] **ЕСИА (Госуслуги)** — способ №2.
- [ ] **ЕБС** — способ №3 (редко нужен).
- [ ] **Собственная система входа (логин/пароль), VK ID, Яндекс ID, банковские ID (Sber ID, T-ID, Alfa ID)** — способ №4 (российский владелец системы).
- [ ] **E-mail (в т.ч. gmail.com) как логин — можно**: это идентификатор внутри российской системы входа, а не иностранный сервис авторизации. (Перспектива: Правительство может потребовать e-mail в зоне .ru — пока не принято.)

### Разбивка по типам размещения

**Облачный портал (SaaS-CRM на домене провайдера)**
- [ ] Авторизацию обеспечивает провайдер (российская система) — базовый вход по телефону/e-mail+пароль соответствует.
- [ ] Проверить и **отключить вход через Google/Apple** в настройках соцсетей-провайдеров, если включён.
- [ ] Домен самого облака — забота провайдера; **кастомный домен**, привязанный к порталу, идентифицировать по Части A.

**On-premise (коробочная система на собственном домене)**
- [ ] Домен → **идентификация по Части A** (если .ru/.рф/.su).
- [ ] В системе входа убрать модули иностранного OAuth/соцсетей (Google/Apple). Оставить телефон/e-mail+пароль/ЕСИА.
- [ ] Локализация ПДн (Часть C): сервер с базой — в РФ.

**Собственный SaaS-сервис**
- [ ] Вход в веб-кабинет/ЛК — российским способом (телефон/e-mail+пароль/банковский ID).
- [ ] Домен на .ru — Часть A; домен на .io/.com и т.п. — 569-ФЗ не применяется.

### Telegram-боты
- [ ] Эксплуатация ботов, общение с клиентами, передача данных в CRM — **не запрещены** (199-ФЗ регулирует авторизацию на сайте/в приложении, не мессенджеры и не CRM).
- [ ] Если бот используется как **вход** в веб-сервис — авторизацией должен считаться российский номер (бот = канал доставки кода), плюс резервный российский способ (SMS/ЕСИА). Зона правовой неопределённости — официального разграничения «канал доставки» vs «система авторизации» нет.

### Существующая база
- [ ] Закон **обратной силы не имеет**. Не удалять разом OAuth и не принуждать действующих пользователей к срочной миграции. Достаточно перевести на российские способы **новые** регистрации/входы и плавно предложить старым привязать российский способ.

## ЧАСТЬ C. Прочие обязательные требования (действуют для всех зон)

### 152-ФЗ — персональные данные (самый «дорогой» блок)
- [ ] **Уведомление РКН** об обработке ПДн подано (нужно почти всем сайтам с формами/аналитикой). Неуведомление — 100–300 тыс. ₽.
- [ ] **Согласие на обработку — отдельным документом** (с 01.09.2025), не «зашитое» в оферту.
- [ ] **Политика обработки ПДн** опубликована; на сайте — наименование/ФИО, ОГРН(ИП)/ИНН, адрес, контакты, **отдельный e-mail для запросов субъектов ПДн** (напр. privacy@домен).
- [ ] **Cookie-баннер** с равноправными «Принять»/«Отклонить»; необязательные куки (аналитика, пиксели, виджеты) не ставятся до выбора.
- [ ] **Локализация баз ПДн в РФ** (ч.5 ст.18). Первичный сбор/хранение данных граждан РФ — на серверах в России. Нарушение — для юрлиц 1–6 млн ₽. Проверить хостинг каждого ресурса.
- [ ] **План на утечку.** Оборотные штрафы (420-ФЗ, с 30.05.2025): 1–3% выручки, до 500 млн при массовой (от 100 тыс. записей). Уведомить РКН в 24 ч, расследование в 72 ч, пострадавших в 10 раб. дней.
- [ ] Учесть автоматический мониторинг РКН (бот сканирует сайты, предписание с 10-дневным сроком).

### 38-ФЗ — реклама
- [ ] Вся интернет-реклама — токен **erid** через ОРД + метка «Реклама» с указанием рекламодателя. До 500 тыс. ₽ за объявление. Касается таргета, контекста, баннеров, нативных интеграций, рекламных рассылок.
- [ ] С 01.09.2026 — расширение отчётности рекламодателей (575-ФЗ).
- [ ] Не размещать рекламу VPN и на нежелательных ресурсах (с 01.09.2025 — штрафы).

### Доступность для инвалидов (ГОСТ Р 52872-2019)
- [ ] **Госорганы/МСУ/подведомственные** — обязательно (ПП РФ № 102 от 07.02.2026, с 01.03.2026, инвалиды по зрению, универсальный дизайн).
- [ ] **Медорганизации** — с 01.03.2026 соответствие ГОСТ по универсальному дизайну (вместо отдельной «версии для слабовидящих»).
- [ ] **Образовательные** организации — в зоне требований по 419-ФЗ.
- [ ] **Коммерческие** сайты — ГОСТ как ориентир; обязателен при госзакупках/работе с госсектором. Минимум: масштабирование текста, контраст ≥4,5:1 (крупный ≥3:1), навигация с клавиатуры, alt к изображениям, понятные ссылки/заголовки.

### Прочее
- [ ] **Реквизиты владельца** на сайте (наименование/ФИО, ОГРН(ИП)/ИНН, адрес, контакты) — ст. 10 ЗоЗПП / 149-ФЗ. Отсутствие — 30–40 тыс. ₽ (ст. 14.5 КоАП).
- [ ] **ЗоЗПП** для интернет-магазинов: инфо о продавце, порядок возвратов, контакты.
- [ ] **Русский язык** на сайте (168-ФЗ, требование с 2026).
- [ ] **Каналы/страницы от 10 тыс. подписчиков** — в реестре РКН.
- [ ] **Хостинг** — только из реестра провайдеров хостинга РКН; провайдер идентифицирует клиента (406-ФЗ).

### Приоритеты по срокам
1. **До 1 сентября 2026** — идентификация всех .ru/.рф/.su доменов (Часть A); снять иностранный OAuth (Часть B).
2. **Уже сейчас (просрочено по закону)** — 152-ФЗ: уведомление РКН, согласия, локализация, cookie-баннер.
3. **Постоянно** — маркировка рекламы (erid), реквизиты, мониторинг изменений (второй/третий антифрод-пакет, требование e-mail в зоне .ru).

> Сводка по открытым источникам и разъяснениям, не юридическое заключение. По доменам с чужими/ликвидированными администраторами и по спорным случаям — сверяться с юристом и правилами конкретного регистратора.
